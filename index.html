SVG vs Icon Fonts

До прихода svg иконочные шрифты справлялись с проблемами масштабирования, стилизации.

			<!-- Добавление иконки на страницу -->
			<i class="fa fa-home" aria-hidden="true"></i>

SVG vs Icon Fonts

  • Частые проблемы с поддержкой и распознаванием шрифта;
  • Разное экранное сглаживание шрифта;
  • Задействие псевдоэлементов ::before и ::after;
  • Стилизация - это только цвет, тень, размер;
  • Сложность в использовании нескольких цветов для иконки;
  • Проблемы при неудачной загрузке, а так же во время загрузки css и шрифта.

SVG vs JPG/PNG

  • Изображения не гибкие - нельзя применить стилизацию;
  • Для современных экраном следует вырезать @2x, @3x и тд картинки;
  • Выбор между хорошим отображением на экране и небольшим весом файла.

Системы координат SVG

The SVG Canvas

Это пространство или область, в которой отображается содержимое SVG. Канвас бесконечен в обоих измерениях. Следовательно, SVG может быть любого размера.

SVG отображается на экране относительно конечной области, известной как viewport. Области SVG, которые лежат за пределами viewport, отсекаются и не видны.

The viewport

Это область просмотра, где содержимое SVG будет видно. Устанавливается атрибутами width и height

			<!-- viewport 100px x 100px -->
			<svg width="100" height="100">...</svg>
Можно задать размер в любых единицах (em, px, pt, cm, percentages, ...)

The viewport

The viewport

Как только задан viewport, браузер устанавливает initial viewport coordinate system и initial user coordinate system

The initial coordinate system

The initial viewport coordinate system — это система координат, установленная для viewport, с началом в левом верхнем углу окна просмотра в точке (0, 0). Ось x направлена вправо, Ось y направлена вниз. Один элемент в исходной системе координат равен одному пикселю в окне просмотра.

The initial coordinate system

The initial user coordinate system — это система координат, установленная на канвасе SVG. Эта система координат изначально идентична системе координат вьюпорта - она ​​имеет начало в верхнем левом углу области просмотра. Атрибут viewBox изменяет исходную систему координат.

The viewBox

viewBox — это «реальная» система координат, используемая для рисования SVG-графики на холсте. Эта система координат может быть меньше или больше, чем viewport.

The viewBox

			<svg viewBox="min-x min-y width height">
				...
			</svg>

min-x, min-y — задают верхний левый угол
width, height — задают ширину и высоту области

The viewBox

В данном примере viewBox равен viewport, но они могут не совпадать

			<svg width="100" height="100" viewBox="0 0 100 100">
				...
			</svg>

The viewBox

Когда viewBox и viewport не совпадают, SVG должно масштабироваться с учетом коэффициента, показывающего разницу этих атрибутов — unit.

1 unit width = viewport-width / viewBox-width
1 unit height = viewport-height / viewBox-height

The viewBox

1 unit = 1

The viewBox

1 unit = 0.5

The viewBox

1 unit = 2

The preserveAspectRatio

Если пропорции ширины и высоты области просмотра и viewBox не совпадают, то артибут preserveAspectRatio указывает браузеру, как отображать рисунок.

Можно провести аналогию с background-position и background-size

The preserveAspectRatio

preserveAspectRatio = defer? <align> <meetOrSlice>?

<align> — состоит из двух частей и задаёт выравнивание viewBox в области просмотра.
<meetOrSlice> — указывает, как пропорции должны быть сохранены (необязательный).

The preserveAspectRatio

preserveAspectRatio="xMaxYMax meet"

Эти значения выравнивают нижний правый угол viewBox по нижнему правому углу области просмотра. meet сохраняет пропорции, масштабируя viewBox, чтобы уместиться в область просмотра насколько это возможно.
